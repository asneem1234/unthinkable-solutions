System Architecture & Design

Deep dive into the technical implementation and ReFRAG methodology

System Overview

The Knowledge-Base Search Engine is built using a modern microservices architecture with distinct service layers for document processing, embedding generation, vector storage, and LLM-based answer synthesis. The system implements Meta AI's ReFRAG methodology for enhanced retrieval and compression.

Architecture Layers

Presentation Layer

Modern responsive UI with vanilla JavaScript for document upload and querying

API Layer

Express.js REST API handling document ingestion and query endpoints

Service Layer

Modular services for processing, embeddings, compression, and RAG operations

Data Layer

Qdrant vector database for semantic search and retrieval

Algorithm Comparison: Standard RAG vs ReFRAG

Understanding the key differences between traditional RAG and Meta AI's ReFRAG methodology that powers this implementation.

Standard RAG

  • Retrieves top-k documents based on similarity
  • Sends all retrieved context to LLM
  • No compression or optimization
  • Higher token consumption
  • Slower processing time
  • Increased API costs
  • May include irrelevant context
  • Simple implementation

ReFRAG (Our Implementation)

  • Retrieves top-k documents intelligently
  • Compresses retrieved context semantically
  • Maintains top-3 most relevant in full
  • 60-70% reduction in tokens
  • 2-3x faster processing
  • Significant cost savings
  • Focused, relevant context only
  • Advanced iterative refinement

Data Flow

Document Ingestion Flow

1️⃣ Upload

User uploads PDF/TXT/DOC files through the web interface

2️⃣ Parse

Document processor extracts text content using appropriate parser

3️⃣ Chunk

Text is split into semantic chunks (500 chars with 50 char overlap)

4️⃣ Embed

Each chunk is converted to vector embeddings using Transformers.js

5️⃣ Store

Embeddings are stored in Qdrant with metadata for retrieval

Query Processing Flow

1️⃣ Query

User submits natural language question through UI

2️⃣ Embed Query

Query is converted to embedding vector for similarity search

3️⃣ Retrieve

Top-10 most similar chunks retrieved from Qdrant

4️⃣ Compress (ReFRAG)

Top-3 kept full, remaining 7 compressed by 90% semantically

5️⃣ Generate

Optimized context sent to Gemini LLM for answer generation

6️⃣ Display

Answer with sources and metrics displayed to user
